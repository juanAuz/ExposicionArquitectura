* Sistemas de Entrada/Salida
La *arquitectura de E/S* de un computador es su interfaz con el
exterior. Esta arquitectura permite una forma sistemática de controlar
su interacción con el mundo exterior a la vez que proporciona al
sistema operativo la información necesaria para para gestionar
perifericos
Para ello, existen *3 técnicas principales*:
#+ATTR_REVEAL: :frag (appear)
- ~E/S Programada~
- ~E/S mediante interrupciones~
- ~Acceso directo a memoria (DMA)~
** Modulo de E/S
Elemento clave de un computador junto al procesador y la
memoria. Permite y gestiona la comunicación entre los *periféricos* y
los *buses del sistema*

~Permite:~
#+ATTR_REVEAL: :frag (appear)
- Realizar la interfaz entre *procesador* y *memoria* a través del bus del sistema
- Realizar la interfaz entre uno o más *perifericos*

#+ATTR_REVEAL: :frag (appear)

** Dispositivos Externos
Los dispositivos externos proporcionan una forma de intercambiar datos
entre el *exterior* y el computador.

~Conexión:~
#+ATTR_REVEAL: :frag (appear)
- Se conectan mediante un *enlace* a un módulo de E/S
- El enlace intercambia señales de *control*, *estado* y *datos*
- También llamados *dispositivos periféricos*
*** Clasificación de dispositivos externos
- *Interacción con humanos*: comunicación con el usuario (VDT, impresoras)
- *Interacción con máquinas*: discos, cintas, sensores, actuadores (robótica)
- *De comunicación*: comunicación con dispositivos remotos
*** Señales de control
Determinan la *función* que debe realizar el dispositivo
#+ATTR_REVEAL: :frag (appear)
- *ENTRADA / LECTURA* ("INPUT" / "READ"): enviar datos al módulo de E/S
- *SALIDA / ESCRITURA* ("OUTPUT" / "WRITE"): aceptar datos desde el módulo de E/S
- Indicar *estado* o ejecutar una *función de control* particular
  ~(ej. posicionar una cabeza de disco)~
*** Señales de estado y datos
#+ATTR_REVEAL: :frag (appear)
- ~Datos~: conjunto de bits enviados o recibidos del módulo de E/S
- ~Estado~: indica la condición del dispositivo
  - *LISTO / NO-LISTO* ("READY/NOT-READY")
  #+ATTR_REVEAL: :frag (appear)
**** ~Componentes internos~
#+ATTR_REVEAL: :frag (appear)
- *Lógica de control*: controla la operación según las órdenes del módulo de E/S
- *Transductor*: convierte señales eléctricas en  otra forma de energía
- *Buffer*: almacena temporalmente el dato en tránsito
  *(tamaño típico: 8 a 16 bits)*

** Teclado/Monitor
Forma más *común* de interacción usuario/computador
#+ATTR_REVEAL: :frag (appear)
- *Teclado*: entrada del usuario
- *Monitor*: muestra entrada y datos del computador
- Unidad básica de intercambio: el *carácter*
**** ~Flujo de la señal~
#+ATTR_REVEAL: :frag (appear)
- *Entrada*: tecla → transductor → código IRA → módulo de E/S
- *Salida*: módulo de E/S → código IRA → transductor → carácter en pantalla
*** Código IRA
#+ATTR_REVEAL: :frag (appear)
- *7 bits* → 128 caracteres posibles ~(similar a ASCII)~
- Dos tipos: *imprimibles* (letras, números) y *de control* (ej. retorno de carro)
- Ejemplo: la letra *K* = ~1001011~

** Controlador de Disco (Disk Drive)
Contiene la electrónica para:
#+ATTR_REVEAL: :frag (appear)
- Intercambiar señales de *datos*, *control* y *estado* con el módulo de E/S
- Controlar el mecanismo de *lectura/escritura* del disco
**** ~Tipos de disco~
#+ATTR_REVEAL: :frag (appear)
- *Cabeza fija*: el transductor convierte patrones magnéticos ↔ bits del buffer
- *Cabeza móvil*: además mueve el brazo *radialmente* sobre la superficie del disco

* Módulos de E/S

~Funciones del módulo de E/S~

Las principales funciones de un módulo de E/S son:

#+ATTR_REVEAL: :frag (appear)
- Control y temporización
- Comunicación con el procesador
- Comunicación con el dispositivo
- Buffer de datos
- Detección de errores


** Comunicación con el procesador

#+ATTR_REVEAL: :frag (appear)
1. El procesador interroga al módulo de E/S
2. El módulo de E/S devuelve el estado del dispositivo
3. El procesador solicita la transferencia del dato
4. El módulo obtiene un dato
5. Los datos se transfieren al procesador

** La comunicación con el procesador implica:
- Decodificación de órdenes
- Información de Estado
- Datos                      


*** Almacenamiento Temporal y detección de errores

*** Buffer de datos

#+ATTR_REVEAL: :frag (appear)
- Compensa diferencias de velocidad
- Almacena temporalmente los datos
- Sincroniza CPU y periféricos

*** Detección de errores

#+ATTR_REVEAL: :frag (appear)
- Fallos del dispositivo
- Errores de transmisión
- Bit de paridad

* E/S Programada

La CPU controla completamente la operación de E/S.

#+ATTR_REVEAL: :frag (appear)
- Envía una orden al módulo de E/S
- Consulta continuamente el estado
- Espera hasta finalizar la operación
- No utiliza interrupciones
#+ATTR_REVEAL: :frag (appear)
*Polling (espera activa)*


** Órdenes de E/S

| Orden     | Función                           |
|-----------+-----------------------------------|
| Control   | Activa o configura el dispositivo |
| Test      | Consulta el estado                |
| Lectura   | Obtiene datos del dispositivo     |
| Escritura | Envía datos al dispositivo        |

* Acceso directo a Memoria DMA
*El DMA (Direct Memory Access)* es un módulo que permite realizar
transferencias de datos entre memoria y dispositivos de E/S sin la
intervención constante del procesador.

Para ello:
- Solicita el control del bus del sistema al CPU.
- Realiza la transferencia de datos.
- Libera el bus cuando termina.

En *Cycle Stealing*, el DMA utiliza temporalmente el bus.
** DMA vs E/S Programada vs E/S mediante interrupción
#+attr_html: :style max-height: 100vh; object-fit: contain;
[[https://cdn.jsdelivr.net/gh/juanAuz/ExposicionArquitectura@main/images/figura-7.4.png]]
** ¿Que información le proporciona el CPU al DMA?
- Tipo de operación
- Dispositivo de Entrada/Salida
- Transferencia de datos
EL DMA detiene al procesador por un ciclo de bus, no lo interrumpe
#+attr_html: :style max-height: 50vh; object-fit: contain;
[[https://cdn.jsdelivr.net/gh/juanAuz/ExposicionArquitectura@main/images/figura_7.12.png]]
** Configuraciones del DMA en el sistema
- *Bus único*: Todos comparten el mismo bus
- *DMA - E/S integrados:* El DMA se conecta directamente a
  los periféricos 
- *Bus E/S propio:* El DMA tiene un bus exclusivo para los
  dispositivos
** Configuraciones del DMA en el sistema
#+attr_html: :style max-height: 100vh; object-fit: contain;

** Controlador Intel 8237 A
- Controlador de DMA de Intel
- Le da acceso directo a la memoria DRAM en computadores con
  procesadores 80x86 
- No almacena ni transporta, solo genera señales de control
- Llamado también controlador fly-by o de vuelo. 
#+attr_html: :style max-height: 30vh; object-fit: contain;
[[https://cdn.jsdelivr.net/gh/juanAuz/ExposicionArquitectura@main/images/inte.jpg]]
** Señales
*Señales HOLD/HOLAD (Controlaodr del DMA - CPU)*
- ~HOLD~: DMA pide el bus
- ~HLDA(Hold Acknowledge)~:Indica que se pueden usar los buses
*Señales DREQ/DACK (Periférico - Controlador del DMA*)
- ~DREQ (DMA Request)~:El periférico solicita el DMA
- ~DACK (DMA Acknowledge)~: El DMA indica que se van a empezar a transferir datos
** Secuencia paso a paso
1. Periférico activa *DREQ*
2. Controlador activa *HRQ* (Hold request) hacia la CPU
3. CPU termina su ciclo de bus y responde con *HLDA*
4. Controlador activa *DACK*
5. Transfiere byte a byte (MEMR/IOW), decrementando el contador
6. Al llegar a cero, desactiva el HRQ y devuelve el bus
** Canales y registros del 8237 a
Dispone de cuatro canales de DMA independientes numerados del 0 al 3
Dispone de 5 registros de control para programar y controlar la
operación del DMA en cada uno de los canales.
